## Issue #49 — Standardize Plaster manifest schema version and shared parameters

**File(s):** `PlasterTemplate/Basic.xml`, `PlasterTemplate/Extended.xml`, `PlasterTemplate/Advanced.xml`

**Problem:** `Basic` and `Extended` use `schemaVersion="1.1"` but `Advanced` uses `schemaVersion="1.0"`. Shared parameters have inconsistent prompts and defaults across the three templates.

**Acceptance Criteria:**
- All three manifests declare `schemaVersion="1.1"`
- Shared parameters have aligned `prompt` text and `default` values (or differences are explicitly documented)
- Integration test (`New-PSNowModule -Template Advanced`) still passes

**Priority:** Medium — cosmetic inconsistency today, but schema drift grows harder to fix as templates evolve.

## Issue #50 — Harden Remove-OldPSNowManifest pre-flight check ⭐ *Delegation candidate*

**File(s):** `Private/Remove-OldPSNowManifest.ps1` (lines 14–27)

**Problem:** `Copy-Item` runs without first verifying the replacement source exists. If `"$BaseManifest.xml"` is missing, the old manifest is deleted and then the copy fails — leaving the template root in a broken, manifest-free state.

**Acceptance Criteria:**
- Function throws a clear terminating error if the source template file does not exist
- The throw happens *before* any `Remove-Item` calls
- New Pester tests cover the missing-template path and verify no `Remove-Item` is invoked

**Priority:** High — data-loss risk (irreversible deletion before error).

## Issue #51 — Write-PSNowStructuredLog: sanitize delimiter characters in field values

**File(s):** `Private/Write-PSNowStructuredLog.ps1`

**Problem:** Field values containing `]`, `,`, `=`, or whitespace are concatenated raw into the log line, producing ambiguous entries that cannot be reliably parsed. No guard against null/empty keys.

**Acceptance Criteria:**
- Values with delimiters are quoted or escaped in output
- Null/empty keys throw a clear `ArgumentException`
- New unit tests cover special-character values, empty-key rejection, null-fields input

**Priority:** Medium — log correctness; becomes important if a log parser is added downstream.

**File(s):** `Private/Write-PSNowStructuredLog.ps1`, `Private/Get-PSNowEnvironmentVariables.ps1`

- `tests/Unit/Get-PSNowEnvironmentVariables.tests.ps1` covers OS detection (mocked), temp path, unsupported-OS error

## Issue #53 — Build tasks Init/Stage should fail fast instead of silently continuing

**File(s):** `Build/build.psake.ps1` (Init ~lines 52–93, Stage ~lines 129–176)

**Problem:** `SilentlyContinue` and unconditional `New-Item` calls can mask setup problems. `Stage` may produce a partial artifact directory without a clear failure signal.

**Acceptance Criteria:**
- `Init` emits a terminating error if required modules are not importable
- `Stage` validates source folders are non-empty before copying
- A deliberate misconfiguration causes `Stage` to abort with a descriptive message

**Priority:** Low — affects developer experience, not end-user functionality.

## Patch Plan: Issue #50 — Remove-OldPSNowManifest Pre-flight Check

This is the recommended first delegation task. It is self-contained (one private function + one test file), has clear success criteria, and fixes a genuine data-loss risk.

### Delegation Brief

> **Goal:** Add a pre-flight guard to `Remove-OldPSNowManifest` that verifies the replacement
> template file exists before touching the current manifest.
>
> **Branch:** `fix/remove-manifest-preflight`
>
> **Files to change:**
> - `Private/Remove-OldPSNowManifest.ps1` — add `Test-Path` check on `$plasterdoc` before the first `Remove-Item`
> - `tests/Unit/Remove-OldPSNowManifest.Logging.tests.ps1` — extend with missing-template test case

### Step-by-step

1. **Read** `Private/Remove-OldPSNowManifest.ps1` lines 23–27.
2. **Insert** before the first `if (Test-Path $upperManifest ...)` block:
   ```powershell
   if (-not (Test-Path -Path $plasterdoc -PathType Leaf)) {
       throw [System.IO.FileNotFoundException]::new(
           "Template source not found: '$plasterdoc'. Cannot replace PlasterManifest.xml.",
           $plasterdoc
       )
   }
   ```
3. **Add test** in `Remove-OldPSNowManifest.Logging.tests.ps1`:
   ```powershell
   It 'throws before deleting when the source template file is missing' {
       Mock Test-Path { $false }   # source template missing
       { Remove-OldPSNowManifest -TemplateRoot '/psnow-test-repo' -BaseManifest 'Basic' } |
           Should -Throw '*Template source not found*'
       Should -Invoke Remove-Item -Exactly 0 -Scope It
   }
   ```
4. **Run** `./Build/Build.ps1 -TaskList analyze,test` — expect all tests green.
5. **Commit** with message: `Fix Remove-OldPSNowManifest: validate source before deleting manifest`
6. **Open PR** referencing `Closes #50`.

### Rollback
Revert the single guard block in `Remove-OldPSNowManifest.ps1`. The added test will then fail, confirming the rollback is in effect.
